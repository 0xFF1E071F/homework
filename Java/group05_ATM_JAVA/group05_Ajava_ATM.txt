大作业group05
ATM取款机
主要任务：
模拟ATM自动柜员机的功能，通过程序来管理银行帐户和用户取款的功能。
具体功能：
1. 当程序运行时，有欢迎的界面。
2. 要求输入卡号，当输入00000时提示，是否进入管理员模式，输入y然后会要求输入管理员密码，默认为000000，
输入正确则进入管理员模式，不正确提示出错并回到欢迎界面。
3. 在管理员模式下，可以对用户的帐户信息进行查询，包括用户的余额和密码和姓名以及历史交易信息，例如输入一个用户的用户账号，然后会出现该用户的相关信息。同时可以对一个已经封存的用户进行解封。
4. 若输入用户名为00001~99999之间则进入普通用户模式，
，提示输入密码，若输入正确则进入普通用户模式，不正确的话提示出错，并要求重新输入密码，假如连续三次依然错误的话，则提示出错并回到欢迎界面。若同时出现两次连续三次出错则自动将该帐户封存。以后若输入此用户的站户则提示该用户的帐户已经被封存，并回到欢迎界面。用户密码默认为000000。
5．进入普通用户模式时，可以实现取款，余额查询，密码修改，转账，和历史交易记录查询，退出，有相关的界面，以及对应选项。
6.用户取款时，取款分为几种。100，200，500，1000，2000，
分别对应1，2，3，4，5。有相关提示，若余额不满足去对应的钱，则对应的提示不出现（例，帐户只有1200元时，则不出现2000元的选项）。取款累计金额不得超过5000元，若超过5000则报告错误，并退到普通用户登录时的欢迎界面。
7余额查询提供现在余额，以及最近两次取款转账的情况。
8．密码修改时，要求第一遍输入的密码为六位数字，若少于六位则提示不足六位，若多于六位则提示多六位，并要求用户重新输入。若第一遍输入无误，则提示输入输入第二遍密码，若与第一遍不符，则从第一遍重新输入，若无误则提示密码修改完成，回到用户初始界面。
9．转帐时出现的的界面与取款类似，功能也相同，只是在选择转账金额以后，出现要求输入转账给的用户的号码的界面，输入两次确认无误，进行转帐。当被转账的用户登录时，会出现转账提示（用户，金额）
10. 历史记录给出最近5次交易记录，取款或是转账，金额是多少。
11.用户退出时，回到开始欢迎界面。
